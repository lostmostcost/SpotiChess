# [기획서] Spotify Chess: 대중문화의 민주화를 위한 데이터 기반 오토배틀러

---

## 1. 프로젝트 개요

- **프로젝트명:** Spotify Chess
- **한줄 정의:** 실제 Spotify 아티스트 데이터를 기반으로 한 '민주적 역밸런싱' 알고리즘 적용 오토배틀러 게임.
- **핵심 가치:** 1%가 독점하는 대중문화의 구조적 문제를 게임 메커니즘으로 해결하고, 유저들이 자발적으로 무명 아티스트를 발굴하도록 유도.
- **기술적 특징:** Python(FastAPI) 기반의 비동기 서버와 Spotify Web API의 실시간 메타데이터 연동.

## 2. 기획 배경 및 문제 정의

### 2.1 대중문화의 '승자 독식' 구조

현재의 대중문화 시장은 소수의 대형 아티스트가 차트와 수익의 대부분을 점유하는 양극화가 심화되어 있습니다. 이는 알고리즘의 '인기 편향성'으로 인해 신예 아티스트가 노출될 기회를 박탈당하는 구조적 문제를 야기합니다.

### 2.2 해결 전략: "약자가 강해지는 게임"

`Spotify Chess`는 이 권력 구조를 뒤집습니다. 게임 내에서 유명한 아티스트는 '힘의 분산' 원칙에 따라 약해지고, 잘 알려지지 않은 아티스트는 잠재력을 폭발시키도록 설계하여 유저가 승리하기 위해 스스로 새로운 음악을 탐험하게 만듭니다.

## 3. 핵심 게임 메커니즘 (Game Design)

### 3.1 아티스트 페르소나 및 유닛 시스템

- **아티스트 페르소나:** 플레이어는 게임 시작 시 한국 아티스트 중 한 명을 '페르소나'로 선택합니다. 이는 유저의 프로필이자 전체적인 시너지의 근간이 됩니다.
- **곡(Track) 유닛:** 상점에서 무작위로 등장하는 곡 카드들이 기물이 됩니다. 각 기물은 Spotify 메타데이터에 근거한 스탯을 가집니다.

### 

### 💡 [수정 제안] 민주화 알고리즘 2.0: 로그(Log) 스케일링

대중문화의 극단적인 수치 격차를 게임의 밸런스(10~100 스탯)로 맞추기 위해 자연로그(ln) 또는 상용로그(log10)를 도입해야 합니다.

**개선된 수식:**

Unit Power=Base Stat×(log10(Followers)log10(Total Streams))×(1−100Selection Rate(%))

**수식 구성 요소 분석:**

1. **언더독 승수 (log10(Followers)log10(Streams)):**
    - 로그를 씌우면 300억과 10만의 격차가 크게 줄어듭니다.
    - 특히, 팔로워(팬덤) 대비 스트리밍(곡의 대중성/실력) 비율이 높은 **'숨겨진 명곡'을 가진 아티스트에게 수학적으로 가장 유리한 가중치**를 줍니다.
2. **메타 페널티 (1−100Selection Rate):**
    - 게임 내에서 픽률이 50%인 유행 덱(메타)은 파워가 절반(0.5)으로 깎입니다.
    - 픽률이 1%인 비주류 덱은 파워 보존율이 99%(0.99)에 달합니다. 0으로 나누는 치명적 오류도 방지합니다.

## 4. 데이터 매핑 전략 (Data Mapping)

Spotify의 `audio-features` 지원 종료에 따라, 기본 제공 메타데이터를 활용한 새로운 스탯 시스템을 제안합니다.

| **Spotify Metadata** | **게임 스탯 / 기믹** | **상세 로직** |
| --- | --- | --- |
| **Duration (곡 길이)** | **HP 및 공격 속도** | 곡이 길면 탱커(높은 HP), 짧으면 어쌔신(빠른 공속) |
| **Genre (장르)** | **시너지 (Synergy)** | K-Pop, Indie, Hip-hop 등 장르별 전용 버프 제공 |
| **Explicit (19금)** | **상태 이상 스킬** | 욕설/자극적 태그가 붙은 곡은 공격 시 적을 '도발'하거나 '방깎' |
| **Release Date** | **성장 곡선** | 최신곡은 초반 화력 집중, 클래식(올드) 곡은 라운드 진행 시 스탯 상승 |
| **Popularity** | **Cost / Rarity** | 인기도가 높을수록 상점에서 높은 가격(Cost)으로 등장 |

### 4. 데이터 매핑 전략 및 스탯 산출 수식 (Data Mapping & Formulas)

Spotify API에서 추출한 메타데이터를 게임 시스템에 맞게 수치화하기 위해 다음과 같은 수학적 모델을 적용합니다. 모든 기물(Unit)은 1성(Star Level 1)을 기준으로 산출되며, 극단적인 메타데이터로 인한 밸런스 붕괴를 막기 위해 기준값(Baseline)을 설정합니다.

### 4.1 체력(HP) 및 공격 속도(AS): 곡 길이(Duration) 기반

Spotify의 `duration_ms`를 초(sec) 단위로 변환하여 사용합니다. 대중음악의 평균 길이인 3분(180초)을 기준점(상수 1.0)으로 삼아, 곡이 길면 체력이 높아지고 짧으면 공격 속도가 빨라지는 반비례 구조를 설계합니다.

- **체력 (HP) 산출 식**
    
    HP=BaseHP×max(0.5,min(1.5,180Duration))
    
    - *설명:* 기준 체력(BaseHP)에 비례하여 체력이 결정됩니다. 곡이 아무리 길거나 짧아도, 스탯 붕괴를 막기 위해 기준 체력의 0.5배에서 1.5배 사이로 상한/하한선(Min/Max Cap)을 둡니다.
- **공격 속도 (Attack Speed) 산출 식**
    
    AS=BaseAS×max(0.6,min(2.0,Duration180))
    
    - *설명:* 단위 시간당 공격 횟수입니다. 2분 미만의 짧은 숏폼 최적화 음원들은 매우 빠른 공격 속도(최대 2배)를 가져 '어쌔신/딜러' 역할을 수행합니다.

### 4.2 기물 등급(Tier) 및 상점 비용(Cost): 인기도(Popularity) 기반

Spotify의 `Popularity` 지수(0~100)를 오토배틀러의 5단계 상점 티어 시스템(1~5 Cost)으로 변환합니다. 올림표기(Ceiling)를 사용하여 직관적으로 매핑합니다.

- **상점 비용 (Cost) 산출 식**
    
    Unit Cost=max(1,⌈20Popularity⌉)
    
    - *설명:* 인기도가 1~20인 마이너 곡은 1코스트, 81~100인 글로벌 메가 히트곡은 5코스트(전설 등급)로 상점에 등장합니다. (인기도가 0인 극단적 경우를 대비해 최솟값을 1로 고정합니다.)

### 4.3 라운드 성장치(Scaling): 발매일(Release Date) 기반

### 4.4 상태 이상 확률: Explicit (19금) 여부

음원의 비속어/선정성 태그인 `Explicit` 플래그는 Boolean(참/거짓) 값이므로, 이를 확률적 상태 이상 부여 메커니즘으로 사용합니다.

- **디버프 발동 확률 (Debuff Chance)**
    
    Proc Chance={10%+(5%×Star Level)0%if Explicit is Trueif Explicit is False
    
    - *설명:* Explicit 곡은 공격 시 일정 확률로 적에게 '출혈'이나 '방어력 감소'를 겁니다. 기물의 성급(1성~3성)이 올라갈수록 디버프 발동 확률이 증가하여 후반부 전투의 변수로 작용합니다.

### 🎮 전투 변수 창출: 장르(Genre) 기반 대분류 시너지 시스템

전투 보드 위에 같은 장르를 가진 기물이 일정 수(예: 2/4/6개) 이상 배치될 때 발동하는 고유 효과입니다. 음악적 특성을 전투 기믹으로 해석하여 예측 불가능한 재미를 창출합니다.

### 1. K-Pop (아이돌/댄스)

- **시너지명:** ⚔️ **칼군무 (Synchronized Attack)**
- **발동 조건:** 3 / 5 기물
- **효과:** 전투 시작 후 매 5초마다, 보드 위의 모든 K-Pop 기물들이 타겟팅을 변경하여 가장 체력이 높은 적 기물 하나를 일제히 집중 공격(Focus Fire)합니다.
- **창출되는 변수:** 상대의 단단한 탱커를 순식간에 녹여버리는 변수를 창출합니다. 유저들에게 '일사불란하게 움직이는 아이돌의 군무'를 연상시킵니다.

### 2. Hip-Hop (랩/힙합)

- **시너지명:** 🎤 **디스전 & 플렉스 (Diss & Flex)**
- **발동 조건:** 2 / 4 기물
- **효과:** 힙합 기물은 적을 처치할 때마다(막타를 칠 때마다) 해당 적이 가지고 있던 남은 기본 공격력의 20/40%를 영구적으로 흡수(Flex)합니다.
- **창출되는 변수:** 초반에 운 좋게 킬을 많이 먹은 1코스트 무명 래퍼 기물이, 후반부에는 5코스트 기물도 1대 1로 이겨버리는 '스노우볼링(Snowballing)' 변수를 만듭니다.

### 3. Rock / Metal (밴드 음악)

- **시너지명:** 🎸 **앙코르 & 모쉬핏 (Encore & Mosh Pit)**
- **발동 조건:** 2 / 4 / 6 기물
- **효과:** 록 기물은 체력이 0이 되어 사망할 때, 즉시 보드에서 사라지지 않고 **최대 체력의 30%로 부활(Encore)하며 5초 동안 공격 속도가 200% 증가하는 광폭화 상태**가 됩니다. (광폭화 종료 후 영구 사망)
- **창출되는 변수:** 다 졌다고 생각한 한타 전투에서, 마지막 남은 록 밴드 기물이 부활하여 폭발적인 딜링으로 상대를 전멸시키는 짜릿한 '역전극'을 연출합니다.

### 4. Indie / Acoustic (인디/어쿠스틱)

- **시너지명:** 🎧 **방구석 아티스트 (Bedroom Artist)**
- **발동 조건:** 2 / 4 기물
- **효과:** 인디 기물은 **자신의 주변 8칸(인접 타일)에 아군 기물이 아무도 없을 때**, 고독 버프를 받아 받는 피해량이 30/50% 감소하고 입힌 피해량의 20%를 체력으로 회복(Lifesteal)합니다.
- **창출되는 변수:** 오토배틀러의 기본인 '뭉쳐서 배치하기' 공식을 깨버립니다. 유저가 인디 덱을 짤 때는 기물들을 보드 구석구석에 흩어지게 배치해야 하는 독특한 공간/배치 변수를 만듭니다.

### 5. EDM / Electronica (일렉트로니카)

- **시너지명:** 🎛️ **드롭 (The Drop)**
- **발동 조건:** 2 / 4 기물
- **효과:** 전투 시작 후 정확히 **7초가 경과하는 순간 (빌드업이 끝나고 드롭이 터지는 순간)**, 보드 전체에 음파가 퍼지며 모든 적 기물이 1.5/3초간 기절(Stun)하고 마법 피해를 입습니다.
- **창출되는 변수:** EDM 덱을 굴리는 유저는 무조건 '7초'를 버티기 위해 탱커 위주의 생존 덱을 구성하게 되며, 7초가 되는 순간 전황이 180도 뒤집히는 쾌감을 줍니다.

### 6. Trot / Traditional (트로트/성인가요)

- **시너지명:** 💸 **행사 뛰기 (Event Tour)**
- **발동 조건:** 2 / 3 기물
- **효과:** 전투 스탯에는 영향을 주지 않지만, **전투에서 승리할 때 살아남은 트로트 기물 1개당 상점 리롤이나 기물 구매에 쓸 수 있는 1 골드(Gold)를 추가로 획득**합니다.
- **창출되는 변수:** '경제(Economy)' 시너지입니다. 전투 자체는 약할 수 있지만, 자본을 빠르게 축적하여 후반부에 고코스트 유닛을 선점하거나 기물 레벨업(3성작)을 빠르게 달성하는 운영의 묘를 살릴 수 있습니다.

### 💡 구현 팁 (데이터 파이프라인 관점)

Spotify는 아티스트 장르를 `["korean pop", "k-pop boy group", "korean r&b"]` 등 매우 파편화된 리스트 형태로 반환합니다. 이를 게임에 적용하기 위해 백엔드(Python) 단에서 다음과 같은 **Mapping Dictionary**를 구성해야 합니다.

Python

`# 예시: 백엔드에서 장르 태그를 대분류로 치환하는 로직
GENRE_MAP = {
    "k-pop": "K-Pop",
    "korean pop": "K-Pop",
    "k-rap": "Hip-Hop",
    "korean r&b": "Hip-Hop", # R&B도 편의상 힙합 분류로 통합 가능
    "korean indie rock": "Rock",
    "k-indie": "Indie",
    "trot": "Trot"
}`

- 가장 먼저 일치하는 키워드를 기준으로 해당 아티스트의 메인 시너지를 1~2개 부여하는 식으로 개발 일정을 단축하고 복잡도를 낮추는 것을 추천해 드립니다.

## 5. 시스템 아키텍처 및 기술 스택

### 5.1 Backend (Python 기반)

- **Framework:** **FastAPI**
    - 선택 이유: 비동기 처리에 최적화되어 Spotify API 호출 및 대량의 전투 로그 연산을 효율적으로 처리 가능.
- **API 연동:** `Spotipy` 라이브러리를 활용하여 실시간 아티스트 정보 동기화.
- **Database:** * **PostgreSQL:** 유저 계정, 덱 정보, 아티스트 메타데이터 저장.
    - **Redis:** 실시간 기물 선택률(Selection Rate) 및 실시간 랭킹 캐싱.

### 5.2 Frontend

- **React + Tailwind CSS:** Spotify 스타일의 다크 모드 UI 구현.
- **Spotify Web Playback SDK:** 기물 배치 및 전투 시 실제 음원 하이라이트 재생.

### Step 1: 데이터 파이프라인 구축 (1~2일차)

- 민주화 알고리즘의 변수 조정을 통해 무명 아티스트의 우승 가능성 검증.
- AWS/Vercel을 통한 최종 배포.
