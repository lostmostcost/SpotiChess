## 📋 프로젝트 개요

- **프로젝트명:** Spotify Chess (6-Hour MVP)
- **개발 목표:** '유명할수록 약하고, 무명일수록 강력해지는' 역밸런싱 로직의 체감형 로그라이크 구현.
- **기술 스택 추천:** Python 단일 스크립트 기반 구동 (웹 UI)
- **데이터 구조:** 외부 API 실시간 호출 대신, 사전에 구성된 로컬 JSON 데이터를 활용하여 개발 시간 단축.

## ▶️ 통합 실행

현재 앱은 FastAPI 백엔드가 프론트엔드 정적 파일과 REST API를 함께 제공하는 단일 프로그램으로 실행됩니다.

```bash
pip install -r requirements.txt
python3 server.py
```

- 웹 UI: `http://127.0.0.1:5173`
- API 상태: `http://127.0.0.1:5173/api`

## 🎮 1. 핵심 게임 루프 및 플로우 가이드

### Phase 1: 페르소나 선택 (Game Start)

1. 게임 실행 시, 준비된 아티스트 JSON 데이터에서 무작위로 **3명의 아티스트**를 프론트엔드(또는 콘솔)에 출력합니다.
2. 플레이어는 이 중 1명의 아티스트를 선택합니다.
3. **[중요]** 선택된 아티스트의 `id`를 저장하며, 이후 게임 내 모든 상점(Shop) 기물은 해당 아티스트의 곡(Track) 리스트 안에서만 100% 무작위(확률 변동 없음)로 등장합니다.

### Phase 2: 상점 및 배치 (Shop & Build)

1. **라운드 시작 처리:** 플레이어에게 기본 수당 **5 Gold**와 기본 경험치 **+2 EXP**를 지급합니다. (1라운드는 예외)
2. **상점 갱신:** 플레이어의 페르소나 아티스트 곡 목록 중 무작위로 **5개**를 뽑아 상점 슬롯에 배치합니다.
3. **플레이어 행동 (제한 시간 없음 - 턴제 기반):**
    - **기물 구매:** 상점의 곡을 클릭하여 구매 후 보드(Board)에 배치합니다. (최대 배치 수는 현재 레벨과 동일)
    - **리롤:** 골드를 지불하고 상점 슬롯 5개를 즉시 새로고침합니다.
    - **경험치 구매:** 골드를 지불하여 즉시 경험치를 올립니다.
    - **기물 판매:** 보드에 있는 기물을 팔고 골드를 전액 돌려받습니다.
4. **전투 돌입:** 플레이어가 '전투 시작' 버튼을 누르면 다음 페이즈로 넘어갑니다.

### Phase 3: 자동 전투 (Combat)

- **적 생성(Enemy Mocking):** 시스템은 현재 라운드 수에 비례하는 체력과 공격력을 가진 '대중음악 차트 괴물(AI)'을 생성합니다.
- **전투 로직 (단순화):**
    - 양측의 기물이 순서대로 기본 공격을 교환합니다.
    - 각 유닛의 HP가 0이 되면 사망 처리됩니다.
    - 15초 이내(또는 일정 턴 이내)에 적의 기물을 모두 전멸시키면 승리, 아군이 전멸하거나 타임아웃 시 패배합니다.

### Phase 4: 라운드 결과 (Resolution)

- **승리 (Win):** 체력 보존, 승리 보너스 **1 Gold** 획득. 다음 라운드(Phase 2)로 이동.
- **패배 (Loss):** 플레이어의 생명력 **1 Heart** 차감.
- **종료 조건 (Game Over / Clear):** 생명력이 0이 되면 즉시 패배 창 출력, 10라운드(임의 설정) 클리어 시 최종 승리 메시지 출력.

## ⚖️ 2. 확정된 밸런싱 수치 (Hardcoded Data)

복잡한 계산 로직을 배제하고 아래의 상수를 그대로 코드에 하드코딩하여 사용합니다.

### 2.1. 경제 (Economy)

- **시작 골드:** 10 Gold (최초 1라운드 시작 시)
- **라운드 기본 수당:** 5 Gold
- **전투 승리 보너스:** 1 Gold
- **기물 구매:** 3 Gold (모든 곡 동일)
- **기물 판매:** 3 Gold (100% 환불)
- **상점 리롤:** 1 Gold
- **경험치 구매:** 4 Gold 지불 시 **4 EXP** 획득

### 2.2. 플레이어 레벨 및 경험치 (Leveling)

보드에 배치 가능한 최대 유닛 수는 플레이어의 레벨과 동일합니다.

(매 라운드 시작 시 자동 +2 EXP 획득)

| **현재 레벨** | **최대 배치 수** | **다음 레벨업 필요 누적 EXP** |
| --- | --- | --- |
| **Lv. 1** | 1개 | 시작 |
| **Lv. 2** | 2개 | 2 EXP |
| **Lv. 3** | 3개 | 8 EXP |
| **Lv. 4** | 4개 | 20 EXP |
| **Lv. 5** | 5개 | 40 EXP |
| **Lv. 6** | 6개 | 72 EXP (Max Level) |

### 2.3. 유닛 스탯 및 역밸런싱 공식 (Stats)

- **기본 체력(HP):** 100 (아군/적군 기물 모두 동일 적용하여 구현 단순화)
- **기본 공격력(Base ATK):** 10
- **최종 공격력 산출 공식 (역밸런싱 적용):**
    
    $$ATK_{final} = 10 \times \left(3 - 2 \times \frac{Popularity}{100}\right)$$
    
    *Popularity(인기도)는 데이터셋에 정의된 0~100 사이의 정수값입니다.*

## 🚀 4. 단계별 구현 가이드 (6시간 마일스톤)

- **[0~1시간] 환경 세팅 및 데이터 준비:**
    - Python 환경 설정 (또는 프론트엔드 환경 세팅).
    - 위 JSON 구조를 바탕으로 5명 정도의 아티스트와 각각의 곡(인기도 포함) 더미 데이터를 생성합니다.
- **[1~3시간] 핵심 클래스 및 로직 구현:**
    - `Player` 클래스: 골드 증감, 체력 관리, 경험치 및 레벨업 메서드.
    - `Shop` 클래스: 특정 아티스트의 `tracks` 리스트에서 `random.sample(tracks, 5)`를 수행하여 상점 갱신. 골드 차감 로직 연동.
    - `Unit` 산출: 역밸런싱 공식을 적용하여 $ATK_{final}$을 계산하는 함수 작성.
- **[3~5시간] 전투 시뮬레이터 연동:**
    - 배치된 아군 유닛의 총합 스탯과 적의 스탯을 비교하여 승패를 결정짓는 단순화된 전투 함수(`def battle():`) 구현.
    - 승패 결과에 따른 골드/체력 반영 로직 연결.
- **[5~6시간] UI 연결 및 폴리싱:**
    - 콘솔에서 텍스트 기반으로 턴이 진행되도록 `while` 루프 작성 또는 Streamlit을 통한 버튼 UI 매핑.
    - 플레이어가 게임의 주제를 이해할 수 있도록 전투 결과 출력 시 **"인기도 {Popularity}의 무명 곡이 무려 {ATK_final}의 데미지를 뿜어냅니다!"** 등의 텍스트 연출 추가.
